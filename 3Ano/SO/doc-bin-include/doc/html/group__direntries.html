◆ soTraversePath()

uint32_t sofs19::soTraversePath ( char *  path)

Get the inode associated to a given path.

The directory hierarchy of the file system is traversed to find an entry whose name is the rightmost component of path.

Parameters
[in]paththe path to be traversed
Remarks
  • path must be absolute and each of its components, with the exception of the rightmost one, must be a directory name or symbolic link name to a path.
  • The process that calls the operation must have traverse (x) permission on all the components of the path with exception of the rightmost one.
  • when calling a function of any layer, use the version with prefix so.
Returns
the corresponding inode number

uint32_t sofs19::soGetDirEntry ( int  pih,
const char *  name 
)

Get the inode associated to a given name.

The directory contents, seen as an array of directory entries, is parsed to find an entry whose name is the given one.

Parameters
[in]pihinode handler of the parent directory
[in]namethe name of the entry to be searched for
Remarks
  • Error EINVAL must be thrown if name is not a base name, that is, if it contains the character '/'.
  • Error ENODIR must be thrown if pih does not represent a directory
  • when calling a function of any layer, use the version with prefix so.
Returns
the corresponding inode number (even if it is NullReference)

void sofs19::soAddDirEntry ( int  pih,
const char *  name,
uint32_t  cin 
)

Add a new entry to the parent directory.

A direntry is added connecting the parent inode to the child inode.

Parameters
[in]pihinode handler of the parent inode
[in]namethe name of the entry to be created
[in]cininode number of the entry to be created
Remarks
  • Assume pih is a valid inode handler of a directory where the user has write access
  • Error EEXIST should be thrown if name already exists
  • Assume cin is a valid inode number
  • The lnkcnt of the child inode is not incremented by this function.
  • when calling a function of any layer, use the version with prefix so.

Delete an entry from a parent directory.

A direntry associated from the given directory is deleted.

Parameters
[in]pihinode handler of the parent inode
[in]namename of the entry
Remarks
  • Assume pih is a valid inode handler of a directory where the user has write access
  • Error ENOENT should be thrown if name does not exist
  • The lnkcnt of the child inode is not decremented by this function.
  • when calling a function of any layer, use the version with prefix so.
Returns
the inode number of the deleted entry
